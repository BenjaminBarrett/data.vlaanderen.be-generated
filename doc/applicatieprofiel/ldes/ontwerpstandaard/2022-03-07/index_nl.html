Afhankelijkheden

Dit document beschrijft een applicatieprofiel, in dit geval OSLO-LDES. Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Overzicht

In dit document wordt correct gebruik van de volgende entiteiten toegelicht:
| Collectie | EventStream | Node | Observatie | onveranderlijkObject | Relatie | RootNode | Versie |

Entiteiten

Collectie

Beschrijving
Een collectie heeft leden die allen voldoen aan een bepaalde shape.
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.

EventStream

Beschrijving
Een Linked Data Event Stream is een collectie van onveranderlijke objecten (zoals versie-objecten & sensor-observaties). Deze objecten volgen op opgelegde shape.
Subklasse van
Collectie
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: lid, node, shape, timestampPath, versionOfPath.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
lid onveranderlijkObject 0..* Een lid maakt deel uit van de collectie.
node Node 0..* Een node is een pagina die de relatie tot andere pagina's beschrijft via een relatie. Via deze relatie kunnen andere nodes gevonden worden.
shape 1 Een bepaalde shape, gedefinieerd door een URI, aan welke alle elementen in de eventstream moeten voldoen. Een shape wordt opgelegd aan alle elementen van de LDES. Voorbeelden van zo'n shape zijn SHACL en ShEx. De shape moet cohorent zijn doorheen heel de LDES.
timestampPath URI 0..1 Geeft weer hoe elementen van een LDES zich tot elkaar verhouden door een tiidstip. Bijvoorbeeld een SHACL property path aan het tijdstip wanneer het versieobject is toegevoegd aan de Linked Date Event Stream.
versionOfPath URI 0..1 Geeft de versie van het element weer. Bijvoorbeeld een SHACL property path naar de niet-geversioneerde identifier van een element.

Node

Beschrijving
Een node is een pagina die de relatie tot andere pagina's beschrijft via een relatie. Via deze relatie kunnen andere nodes gevonden worden.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: node, retentiebeleid.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
node Node 0..* Een node is een pagina die de relatie tot andere pagin's beschrijft via een relatie. Via deze relatie kunnen andere nodes gevonden worden.
retentiebeleid Retentiebeleid 0..1 Een retentiebeleid is een beleid volgens dewelke elementen (i.e. data) verwijdert zullen worden. Een retentiebeleid KAN gebruikt worden door een data-leverancier wanneer hij/zij niet meer alle events wenst aan te bieden.

Observatie

Beschrijving
Het vaststellen van de waarde van een bepaald kenmerk van een Object op een bepaald tijdstip of tussen twee tijdstippen.
Gebruik
Het gaat om schattingen van de waarde,maw op de waarde zit een zekere foutmarge. Voorbeelden zijn temperatuur,klasse,kleur. Dit is verschillend van waarden die zijn toegekend en dus geen schattingen zijn zoals naam of prijs.
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.

onveranderlijkObject

Beschrijving
Een onveranderlijkObject is het basiselement van een Linked Data Event Stream.
Gebruik
Een OnveranderlijkObject maakt deel uit van een Linked Data Event Stream en wordt beschreven d.m.v. RDF tripples. Het OnveranderlijkObject moet voldoen aan de shape vereisten van de LDES.
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.

Relatie

Beschrijving
Een klasse die de klasse tussen twee nodes beschrijft
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: Node (source), Node (target), path, value.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
Node (source) Node 1 Referentie naar verbonden klasse.
Node (target) Node 1 Referentie naar verbonden klasse.
path URI 1 TO DO TO DO
value URI 1 TO DO TO DO

RootNode

Beschrijving
Een RootNode is een beginpagina, die zelf al leden kan bevatten, via wiens gelinkte paginas alle leden kunnen gevonden worden.
Gebruik
een tree:Node die verwezen wordt vanuit een ldes:EventStream via een tree:view property
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.

Versie

Beschrijving
Stelt een entiteit voor in één specifieke periode.
Eigenschappen
Voor deze entiteit zijn geen eigenschappen gedefinieerd.
